1.2.1. Què és un framework de costat client

Un framework és una eina per fer apps web sense recarregar la pàgina. S'executa al navegador i gestiona la interfície de manera dinàmica.

    1.2.2. Per què necessitem frameworks

    Problema: Abans havíeu de manipular el DOM manualment per cada canvi. Molt codi repetitiu, difícil de mantenir.

    Solució: Els frameworks automatitzen això amb components reutilitzables. Declareu com voleu que es vegi la interfície i ells gestionen les actualitzacions.

    1.2.3. Quan usar frameworks

    • Apps amb molta interacció: Filtres, cerca, actualitzacions en temps real
    • Projectes que creixeran: Començar amb estructura clara evita reescriure
    • Equips grans: Convencions comunes faciliten col·laboració
    • Experiència tipus app nativa: Navegació fluida sense recàrregues

    1.2.4. Arquitectura client/servidor

    En una app moderna:

    Servidor:

    • Dades: emmagatzema a BD
    • Lògica: validacions, càlculs
    • Seguretat: autenticació, permisos
    • API: exposa endpoints

    Client (amb framework):

    • Presentació: mostra dades amb HTML/CSS
    • Interacció: captura clics, tecles
    • Navegació: canvia vistes sense recarregar
    • Estat local: recorda filtres, seleccions

    Aquesta separació permet que backend i frontend treballin en paral·lel.

    1.5.2. Les 4 fases

    Fase 1: Pàgines estàtiques (anys 90)

    • HTML escrit a mà
    • Cap interactivitat
    • Cada pàgina = fitxer independent

    Fase 2: Pàgines dinàmiques al servidor (finals anys 90 - anys 2000)

    • PHP, ASP generen HTML segons dades
    • JavaScript mínim al client
    • Limitació: cada clic recarrega tot

    Fase 3: AJAX (mitjans anys 2000)

    • AJAX permet actualitzar parts sense recarregar
    • jQuery facilita manipular DOM
    • Limitació: codi desorganitzat en créixer

    Fase 4: Frameworks moderns (anys 2010 - actualitat)

    • Angular, React, Vue introdueixen components
    • SPA: carreguen tot un cop, navegació local
    • Experiència similar a apps natives

    Clau: Els frameworks són la resposta natural a la complexitat creixent de les apps web.

    1.6.2. Definició

    Un framework proporciona:

    • Components: peces reutilitzables (botó, llista, formulari)
    • Plantilles: sintaxi per descriure HTML dinàmic
    • Estat: sincronitza dades entre components
    • Rutes: navegació sense recarregar
    • CLI: eines per crear projectes i desenvolupar

    1.6.3. Framework vs Llibreria

    • Llibreria: Crideu funcions quan vulgueu (jQuery)
    • Framework: Defineix l'estructura i crida el vostre codi (inversió de control)

    1.6.4. Funcions principals

    1. Renderitzar interfície: Components generen HTML dinàmic
    2. Gestionar estat: Mantenir dades sincronitzades
    3. Enrutament: URLs sense recarregar (/productes, /carret)
    4. Serveis HTTP: Comunicar amb API del servidor

    Clau: Framework = estructura + eines + ecosistema.

    1.7.2. Els 4 grans

    Angular (Google, 2010)

    • Complet: tot inclòs (rutes, HTTP, formularis)
    • TypeScript obligatori
    • Opinió forta: una manera correcta de fer-ho
    • Ideal per: projectes grans, equips, educació

    React (Meta/Facebook, 2013)

    • Llibreria enfocada en la vista
    • JSX: HTML dins JavaScript
    • Flexible: trieu llibreries externes
    • Ideal per: projectes de tot tipus, més popular

    Vue (comunitat, 2014)

    • Progressiu: useu el que necessiteu
    • Sintaxi clara i fàcil d'aprendre
    • Balanç entre Angular i React
    • Ideal per: prototips ràpids, equips petits

    Svelte (comunitat, 2016)

    • Compila a JavaScript optimitzat
    • Sense runtime virtual DOM
    • Menys codi, més rendiment
    • Ideal per: apps amb restriccions de pes

    1.7.3. Per què triem Angular al curs?

    • Estructura clara: Aprendre patrons des del principi
    • Tot inclòs: No cal triar 10 llibreries externes
    • TypeScript: Detecció d'errors abans d'executar
    • Estabilitat: Versions predictibles, suport Google

    Clau: Angular és ideal per aprendre fonaments abans d'explorar altres opcions.
