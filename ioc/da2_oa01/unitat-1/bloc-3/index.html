1.1. Prerequisits

Abans de començar aquesta lectura, assegureu-vos que heu completat:

  • Lectura 1.1.3 — Instal·lació de Node.js i npm: Node.js (versió 18.x o 20.x) i npm funcionen correctament.
  • Lectura 1.2.1 — Instal·lació i configuració de l'IDE: VS Code està operatiu amb la terminal integrada.
  • Lectura 1.2.2 — Control de versions amb Git: Git està instal·lat i configurat.

A més, necessiteu:

  • Permisos per instal·lar paquets globals: capacitat d'executar npm install -g o configuració d'un prefix d'usuari.
  • Connexió a Internet: per descarregar l'Angular CLI (aproximadament 40-50 MB).
  • Espai de treball preparat: carpeta ~/workspace o %USERPROFILE%\workspace per provar la CLI.
  • Terminal operativa: PowerShell (Windows), zsh o bash (macOS/Linux).

Verificació ràpida:

BASH

Punt d'autocomprovació: Node.js i npm retornen versions correctes sense errors. Angular CLI depèn completament d'aquestes eines.

1.2. Marc conceptual

1.2.1. Què és una CLI

Una CLI (Command Line Interface) és un programa que s'executa des del terminal i permet interactuar amb un sistema mitjançant comandes de text.

Avantatges CLI vs GUI:

  • Més ràpida: No cal navegar per menús visuals
  • Automatitzable: Scripts per repetir tasques
  • Més precisa: Opcions avançades no disponibles en GUI

1.2.2. Per què els frameworks tenen CLI

Problema sense CLI:

BASH 
# Crear component manualment (5 fitxers):
mkdir src/app/components/targeta
touch src/app/components/targeta/targeta.component.ts
touch src/app/components/targeta/targeta.component.html
touch src/app/components/targeta/targeta.component.scss
touch src/app/components/targeta/targeta.component.spec.ts
# Escriure tot el codi boilerplate...
# Registrar al module...
# Propens a errors!

Solució amb CLI:

BASH 
ng generate component targeta
# Crea 4 fitxers en 2 segons, tot configurat!

1.2.3. Tasques que automatitza

1. Crear projectes:

BASH 
ng new cataleg-elements
# Genera estructura completa:
# - src/ amb codi font
# - angular.json configuració
# - package.json dependències
# - tsconfig.json TypeScript
# - .git/ repositori Git
# - 100+ fitxers en 30 segons!

2. Generar codi:

BASH 
ng generate component header
ng generate service data
ng generate pipe formatDate
ng generate guard auth
# Plantilles estandarditzades, menys errors

3. Compilar:

BASH 
ng build --configuration production
# TypeScript → JavaScript
# SCSS → CSS
# Minificat, optimitzat, llest per desplegar

4. Servir aplicacions:

BASH 
ng serve
# Servidor local: http://localhost:4200
# Hot reload: canvis es veuen a l'instant
# No cal refrescar navegador manualment!

5. Executar tests:

BASH 
ng test
# Llança Karma + Jasmine
# Proves unitàries automàtiques

6. Actualitzar:

BASH 
ng update @angular/core @angular/cli
# Actualitza Angular amb migracions automàtiques

1.2.4. Avantatges d'usar CLI oficial

Estalvi de temps:

  • Component manual: 10-15 minuts
  • Component amb CLI: 2 segons

Coherència:

BASH 
# Tots els components segueixen mateix patró:
ng g component user-profile
ng g component product-list
ng g component shopping-cart
# Estructura idèntica, fàcil navegar projecte

Menys errors:

  • CLI genera codi sintàcticament correcte
  • No oblides importar
  • No escrius malament els decoradors

Millors pràctiques:

  • Incorpora patrons recomanats per Angular
  • Estructura organitzada
  • Configuració òptima

Integració:

  • Totes les eines funcionen juntes
  • No cal configurar webpack, babel, etc.
  • "Funciona out of the box"

1.2.5. Angular CLI específicament

Característiques:

  • Projecte complet amb 1 comando: ng new
  • 40+ esquemes de generació: components, serveis, pipes, guards, interceptors...
  • Servidor de desenvolupament: hot reload integrat
  • Compilació optimitzada: tree-shaking, minificació, compressió
  • Tests integrats: Karma, Jasmine, Protractor
  • Actualitzacions guiades: ng update amb migracions

1.2.6. Comparativa: manual vs CLI

Crear component 10-15 minuts 2 segons
Compilar producció Config webpack complex ng build --prod

1.3.7. Conceptes clau

  • Global vs local: CLI s'instal·la global (-g), projectes són locals
  • Hot reload: Canvis al codi es veuen automàticament sense refrescar
  • Standalone: Components sense NgModules (simplificat)
  • Esquema: Plantilla de codi que genera la CLI
  • Dry run: Simular comanda sense fer canvis (--dry-run)
  • Telemetria: Dades anònimes d'ús (es pot desactivar)

Aquest model mental us ajuda a entendre el paper de la CLI en el desenvolupament Angular.

1.4. Què és una CLI i per què els frameworks en tenen

1.4.1. Objectiu

Entendre què és una CLI i per què és essencial per desenvolupament modern.

1.4.2. Definició CLI

CLI (Command Line Interface): Programa que s'executa al terminal amb comandes de text.

Exemples de CLI:

  • git (control versions)
  • npm (gestor paquets)
  • ng (Angular)
  • docker (contenidors)

1.4.3. Per què CLI són útils

Avantatges:

  • Velocitat: Més ràpid que GUI
  • Automatització: Scripts i CI/CD
  • Precisió: Més opcions disponibles
  • Documentació: Facil buscar ajuda online

1.4.4. Per què frameworks tenen CLI

Problemes que resolen:

1. Configuració complexa:

  • Sense CLI: configurar webpack, babel, TypeScript, linters... (hores)
  • Amb CLI: tot preconfigurats

2. Generar codi repetitiu:

  • Components, serveis... sempre mateixa estructura
  • CLI genera automàticament

3. Mantenir coherència:

  • Tots els components segueixen mateix patró
  • Facilita treballar en equip

4. Actualitzacions:

  • Migrar entre versions és complex
  • CLI automatitza amb ng update

1.4.5. Exemples d'automatització

Sense CLI:

  • Crear 10 components: 2 hores
  • Configurar build: 4 hores
  • Setup tests: 2 hores

Amb CLI:

  • Crear 10 components: 20 segons (ng g c × 10)
  • Configurar build: inclòs
  • Setup tests: inclòs
Informació
Clau: CLI automatitza tasques repetitives, garanteix coherència i estalvia hores de configuració.

1.4.6. Resultat esperat

Compreneu per què frameworks moderns proporcionen CLI oficial.

1.8. Exploració de comandes disponibles

1.8.1. Objectiu

Conèixer les comandes principals que usareu.

1.8.2. ng new

Funció: Crea projecte Angular complet

Sintaxi:

BASH 
ng new <nom-projecte> [opcions]

Exemple:

BASH 
ng new cataleg-elements --standalone --routing --style=scss

Opcions principals:

  • --standalone: Components autònoms (recomanat)
  • --routing: Configura rutes
  • --style=scss: Preprocessador CSS
  • --skip-git: No crear repositori Git
  • --skip-install: No instal·lar dependències

1.8.3. ng generate (ng g)

Funció: Genera codi (component, service, etc.)

Sintaxi:

BASH 
ng generate <esquema> <nom> [opcions]

Exemples:

BASH 
ng g component header
ng g service data
ng g pipe uppercase
ng g guard auth
ng g interface user

Opcions:

  • --skip-tests: No crear fitxer test
  • --flat: No crear carpeta pròpia
  • --dry-run: Simular sense crear

1.8.4. ng serve (ng s)

Funció: Servidor desenvolupament

Sintaxi:

BASH 
ng serve [opcions]

Exemple:

BASH 
ng serve --open --port=4300

Opcions:

  • --open (-o): Obrir navegador automàtic
  • --port=4300: Canviar port (defecte: 4200)
  • --poll=2000: Verificar canvis cada 2s (si hot reload falla)

1.8.5. ng build (ng b)

Funció: Compila aplicació

Sintaxi:

BASH 
ng build [opcions]

Exemple:

BASH 
ng build --configuration production

Opcions:

  • --configuration production: Optimitzacions producció
  • --output-path=dist: Canviar directori sortida

1.8.6. ng test (ng t)

Funció: Tests unitaris

Sintaxi:

BASH 
ng test [opcions]

Opcions:

  • --watch=false: Executar un cop (no mode watch)
  • --code-coverage: Generar informe cobertura
Informació
Clau: Aquestes 5 comandes són el 90% del que usareu diàriament.

1.8.7. Resultat esperat

Coneixeu sintaxi i opcions de les 5 comandes principals.

1.11.1. Objectiu

1.11.2. Flux complet

1. Crear projecte (1 cop):

2. Llançar servidor (desenvolupament diari):

BASH

3. Generar components (desenvolupament diari):

BASH

4. Editar codi (desenvolupament diari):

5. Verificar canvis (desenvolupament diari):

BASH

7. Tests (abans de desplegar):

BASH 
ng test

8. Compilar producció (desplegar):

BASH 
ng build --configuration production

9. Desplegar (publicar):

Pujar carpeta dist/ a servidor

1.11.3. Comandes més usades

Diari:

  • ng serve (executar servidor)
  • ng g component (crear components)

Setmanal:

  • ng build (compilar)
  • ng test (tests)

Ocasional:

  • ng new (crear projecte nou)
  • ng update (actualitzar)

1.11.4. Avantatges del flux

  • Tot des de terminal
  • No cal canviar d'aplicació
  • Hot reload automàtic
  • CLI automatitza configuracions

1.17. Repàs de conceptes
  • CLI (Command Line Interface): Eina de terminal que s'executa amb comandes de text.
  • Angular CLI: Eina oficial per desenvolupar aplicacions Angular, automatitza desenvolupament.
  • npm install -g: Instal·lació global d'un paquet npm, accessible des de qualsevol ubicació.
  • Prefix global: Directori on npm instal·la paquets globals (exemple: ~/.npm-global).
  • PATH: Variable d'entorn que defineix on el sistema busca executables.
  • Comandes principals:
    • ng new <nom>: Crea projecte Angular complet.
    • ng generate <esquema> <nom> (ng g): Genera components, serveis, pipes, guards, etc.
    • ng serve (ng s): Servidor desenvolupament amb hot reload.
    • ng build (ng b): Compila aplicació per producció.
    • ng test (ng t): Tests unitaris.
    • ng help: Ajuda comandes.
    • ng version (ng v): Versió CLI i entorn.
  • Opcions habituals:
    • --standalone: Components autònoms (sense NgModules).
    • --routing: Configura rutes.
    • --style=scss: Format styles (css, scss, sass, less).
    • --skip-tests: No genera fitxers test.
    • --open (-o): Obre navegador automàtic.
    • --port=<número>: Canvia port servidor.
    • --configuration=production: Compila amb optimitzacions.
    • --dry-run: Simula sense fer canvis.
  • Telemetria: Dades anònimes d'ús per Angular. Desactivar: ng analytics off.
  • Gestor paquets: Eina per instal·lar dependències (npm, yarn, pnpm). Fixar: ng config -g cli.packageManager npm.
  • Hot reload: Recàrrega automàtica navegador quan es detecten canvis durant ng serve.
  • Standalone components: Arquitectura Angular (v14+) on components no necessiten NgModules.
  • SCSS: Preprocessador CSS amb variables, nesting, mixins per styles mantenibles.

Cada desenvolupador configura diferent: uns usen Webpack, altres Rollup; uns TypeScript strict, altres no. Això dificulta col·laboració. ng new crea estructura estandarditzada que tots els desenvolupadors Angular reconeixen immediatament.

4. Actualitzacions complexes

Actualitzar Angular manualment requereix canviar desenes de fitxers de configuració, actualitzar dependències compatibles i migrar codi. ng new + ng update automatitza aquest procés amb migracions guiades.

3.2. Marc conceptual

3.2.1. Què és un servidor de desenvolupament

Un servidor de desenvolupament és una eina que executa la vostra aplicació en un entorn local (el vostre ordinador) mentre desenvolupeu. Compila el codi, el serveix a través d'un servidor HTTP local i detecta canvis automàticament per actualitzar el navegador.

3.2.2. Problema: desenvolupar sense servidor de desenvolupament

Situació abans dels servidors de desenvolupament moderns:

TEXT 
1. Editar codi TypeScript
2. Executar compilador manualment: tsc
3. Esperar compilació (30 segons)
4. Obrir/refrescar navegador manualment
5. Veure error al navegador
6. Tornar a l'editor
7. Corregir error
8. REPETIR PASSOS 2-6 cada canvi (100+ vegades al dia)

→ Cada cicle: 1-2 minuts
→ 100 canvis al dia = 2-3 hores perdudes només refrescant i recompilant

Situació amb servidor de desenvolupament:

TEXT 
1. Arrencar servidor un cop: ng serve
2. Editar codi
3. Desar (Ctrl+S)
4. Navegador s'actualitza automàticament (1 segon)
5. Veure resultat immediatament

→ Cada cicle: 1-3 segons
→ 100 canvis al dia = 5 minuts totals

3.2.3. Problemes que resol el servidor de desenvolupament

1. Refrescar navegador manualment

Sense servidor: després de cada canvi, cal anar al navegador (Alt+Tab), refrescar (F5), tornar a l'editor. Amb 100 canvis diaris, són 100 interrupcions del flux de treball. El servidor detecta canvis i actualitza automàticament el navegador via WebSocket.

2. Compilació manual

TypeScript no s'executa directament al navegador: cal compilar a JavaScript. SCSS tampoc: cal compilar a CSS. Sense servidor, heu d'executar tsc i compiladors manualment després de cada canvi. El servidor ho fa automàticament i només recompila els fitxers canviats (compilació incremental).

3. Errors difícils de diagnosticar

Sense servidor, errors de compilació apareixen com text críptic al terminal sense context. El servidor mostra errors al navegador amb ubicació exacta del fitxer, línia, columna i suggeriments de correcció.

4. No poder usar imports moderns

Els navegadors antics no entenen imports ES6 (import { Component } from '@angular/core') directament des del sistema de fitxers (protocol file://). Cal un servidor HTTP per resoldre mòduls correctament. El servidor serveix l'aplicació via HTTP amb bundling adequat.

5. Cicle de desenvolupament lent

Cada canvi requereix: compilar → copiar fitxers → refrescar → esperar càrrega. Aquest cicle suma minuts diaris. El servidor optimitza tot el procés amb compilació incremental (només canvis), caché intel·ligent i hot reload instantani.

3.2.7. Característiques principals del servidor

Hot Module Replacement (HMR):

Actualitza mòduls JavaScript al navegador sense recarregar la pàgina completa. Preserva estat de l'aplicació (dades de formularis, scroll position) quan és possible.

Watch mode (mode observació):

Monitora constantment tots els fitxers de src/. Detecta canvis en milisegons (inotify a Linux, FSEvents a macOS, ReadDirectoryChangesW a Windows). Inicia recompilació automàtica només dels fitxers afectats.

Compilació incremental:

Només recompila fitxers modificats i dependències directes. Primera compilació: 10-30 segons. Recompilacions: 1-3 segons. Caché intel·ligent a .angular/cache/.

Servidor HTTP integrat:

Servidor Express.js a localhost:4200. Serveix fitxers compilats des de memòria (no disc). Gestiona CORS, proxies, rutes, recursos estàtics.

WebSocket per comunicació:

Canal bidireccional entre servidor i navegador. Servidor notifica navegador quan hi ha canvis compilats. Navegador envia confirmació de recepció.

Missatges d'error rics:

Errors de compilació TypeScript al navegador amb ubicació exacta. Errors runtime amb stack traces detallats. Warnings visibles al terminal i navegador.

3.17. Repàs de conceptes
  • Servidor de desenvolupament: eina que compila i serveix l'aplicació localment amb hot reload automàtic.
  • ng serve: comanda CLI per arrencar el servidor de desenvolupament.
  • localhost: nom del vostre ordinador; equivalent a 127.0.0.1.
  • Port 4200: port per defecte del servidor Angular.
  • Hot reload: recàrrega automàtica del navegador quan detecta canvis als fitxers.
  • Compilació incremental: només recompila fitxers modificats (1-3 seg vs 30 seg completa).
  • Watch mode: servidor observa fitxers constantment per detectar canvis.
  • WebSocket: canal comunicació servidor-navegador per notificar actualitzacions.
  • Ctrl+C: combinació per aturar el servidor en tots els sistemes operatius.
  • Compiled successfully: missatge que indica compilació completada sense errors.
  • localhost vs producció: localhost només vosaltres, producció Internet públic.
