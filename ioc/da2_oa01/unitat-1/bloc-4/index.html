Introducció

Contingut multimèdia d'aquesta lectura

En aquest cas d'ús pràctic, integrareu tots els conceptes treballats a la Unitat 1 per preparar l'entorn complet de desenvolupament Angular. Configurareu les eines necessàries, creareu el vostre primer projecte i el publicareu a GitHub, establint així les bases per a tot el curs.

Aquest cas d'ús representa un flux de treball realista que us permetrà aplicar els coneixements adquirits sobre instal·lació d'eines, configuració de l'entorn, generació de projectes i control de versions en un context integrat.

Objectius

En finalitzar aquest cas d'ús, hauríeu de ser capaços de:

  • Verificar i configurar l'entorn tècnic complet (Node.js, npm, Angular CLI, VS Code, Git)
  • Generar el projecte base cataleg-elements amb estructura funcional
  • Validar servidor de desenvolupament amb hot reload operatiu
  • Publicar el projecte a GitHub amb control de versions actiu
  • Documentar el procés i preparar el projecte per a les següents unitats

Criteris d'avaluació (RA1)

Aquest cas d'ús cobreix els següents criteris del Resultat d'Aprenentatge 1:

  • RA1.1: Identifica les eines imprescindibles i comprova que compleixen els requisits del sistema
  • RA1.2: Instal·la i configura l'entorn de desenvolupament amb les eines i dependències necessàries
  • RA1.3: Crea l'estructura base del projecte organitzant carpetes i fitxers de manera coherent
  • RA1.4: Configura els fitxers d'execució i comprova que l'entorn arrenca correctament
  • RA1.5: Verifica el funcionament global amb proves o exemples senzills

1. Prerequisits i verificació inicial

Abans de començar, necessiteu:

  • Node.js 18.x o 20.x instal·lat
  • npm 9.x o 10.x disponible
  • Git 2.x configurat
  • Visual Studio Code amb extensions: Angular Language Service, ESLint, Prettier
  • Compte GitHub creat i verificat
  • Carpeta de treball: ~/workspace (Linux/macOS) o %USERPROFILE%\workspace (Windows)

Script de verificació ràpida (5'):

BASH 
# Linux/macOS
cd ~/workspace && node --version && npm --version && ng version | head -n 3 && git --version && code --version
POWERSHELL 
# Windows PowerShell
Set-Location $env:USERPROFILE\workspace; node --version; npm --version; ng version | Select-String "Angular CLI"; git --version; code --version
Èxit
Punt de control: Totes les comandes responen amb versions. Si alguna falla, instal·leu l'eina abans de continuar.

4. Configuració Git i GitHub

4.1. Objectiu

Configurar Git amb identitat global i crear compte GitHub funcional per al control de versions remot.

4.2. Context d'integració

Aplica conceptes de Lectures 1.2.2 (Git local) i 1.2.3 (GitHub remot).

4.3. Pas 1: Configurar identitat Git

BASH 
# Configuració global
git config --global user.name "El Vostre Nom"
git config --global user.email "el.vostre.email@example.com"

# Verificar configuració
git config --global --list

Verificació:

BASH 
git config --global user.name
git config --global user.email

Resultat: Mostra el vostre nom i email.

4.4. Pas 2: Crear repositori de prova local

BASH 
# Linux/macOS
cd ~/workspace
mkdir prova-git && cd prova-git
git init

# Windows PowerShell
Set-Location $env:USERPROFILE\workspace
New-Item -ItemType Directory -Name prova-git
Set-Location prova-git
git init

Crear fitxer de prova:

BASH 
echo "# Repositori de prova Git" > README.md
git add README.md
git commit -m "feat: afegir README inicial"

Verificació:

BASH 
git log --oneline

Resultat: Mostra 1 commit amb el missatge "feat: afegir README inicial".

4.5. Pas 3: Crear compte GitHub

  1. Aneu a https://github.com/signup
  2. Completeu registre amb email, contrasenya i nom d'usuari
  3. Verifiqueu el correu electrònic (reviseu safata d'entrada i spam)

Verificació: Inicieu sessió a https://github.com i veieu el vostre perfil.

4.6. Pas 4: Crear repositori remot de prova

  1. A GitHub, cliqueu New repository (botó verd dalt a la dreta)
  2. Repository name: prova-git
  3. Description: "Repositori de prova per verificar Git i GitHub"
  4. Deixeu Public seleccionat
  5. NO marqueu "Add a README file" (ja el teniu)
  6. Cliqueu Create repository

Connectar local amb remot:

BASH 
# Des de ~/workspace/prova-git (o %USERPROFILE%\workspace\prova-git)
git remote add origin https://github.com/EL-VOSTRE-USUARI/prova-git.git
git branch -M main
git push -u origin main

Si demana credencials:

  • Username: el vostre usuari GitHub
  • Password: token d'accés personal (PAT) o contrasenya (deprecated)

Verificació: Aneu a https://github.com/EL-VOSTRE-USUARI/prova-git i veieu el README.md publicat.

4.7. Troubleshooting

Problema Causa Solució
git: command not found Git no instal·lat Linux: sudo apt install git. macOS: brew install git. Windows: Descarregueu des de https://git-scm.com.
Email no verificat a GitHub No heu clicat l'enllaç Reviseu la safata d'entrada i spam. Reenviï l'email des de GitHub si cal.
git push demana credencials contínuament HTTPS sense cache Configureu cache: git config --global credential.helper cache (Linux/macOS) o git config --global credential.helper wincred (Windows).
Error remote: Permission denied Credencials incorrectes Creeu token PAT a GitHub (Settings → Developer settings → Personal access tokens) i utilitzeu-lo com a contrasenya.

4.8. Checkpoint

Git configurat amb nom i email globals
Repositori local prova-git inicialitzat amb 1 commit
Compte GitHub creat i email verificat
Repositori remot prova-git creat a GitHub
git push funciona i README.md és visible a GitHub

5. Generació del projecte Angular

5.1. Objectiu

Generar el projecte cataleg-elements amb Angular CLI i estructura preparada per al desenvolupament.

5.2. Context d'integració

Aplica conceptes de Lectures 1.3.1 (Angular CLI) i 1.3.2 (estructura projecte).

5.3. Pas 1: Generar projecte amb ng new

BASH 
# Linux/macOS
cd ~/workspace
ng new cataleg-elements --routing --style=scss --ssr=false

# Windows PowerShell
Set-Location $env:USERPROFILE\workspace
ng new cataleg-elements --routing --style=scss --ssr=false

Opcions durant la generació:

  • Standalone components? → Yes (Enter)
  • Install dependencies? → Yes (Enter)

Espera: La instal·lació de dependències pot trigar 3-5 minuts.

Verificació:

BASH 
cd cataleg-elements
ls -la    # Linux/macOS
dir       # Windows

Resultat: Veieu carpetes src/, node_modules/, fitxers angular.json, package.json, etc.

5.4. Pas 2: Explorar estructura generada

Obriu el projecte a VS Code:

BASH 
code .

Carpetes principals:

  • src/app/ → Components i codi de l'aplicació
  • src/assets/ → Imatges, fonts, recursos estàtics
  • node_modules/ → Dependències npm (NO pujar a Git)

Fitxers clau:

  • angular.json → Configuració Angular CLI
  • package.json → Dependències i scripts npm
  • tsconfig.json → Configuració TypeScript
  • .gitignore → Fitxers exclosos de Git

Verificació: A VS Code, explorador lateral mostra totes aquestes carpetes i fitxers.

5.5. Pas 3: Verificar package.json

Obriu package.json i confirmeu dependències:

JSON 
{
  "dependencies": {

Verificació: Versió Angular és 18.x.x.

5.6. Pas 4: Crear carpetes preparatòries

Des del terminal integrat de VS Code (Ctrl+`):

BASH 
# Linux/macOS/Git Bash
cd src/app
mkdir components services models pages

# Windows PowerShell
Set-Location src\app
New-Item -ItemType Directory -Name components
New-Item -ItemType Directory -Name services
New-Item -ItemType Directory -Name models
New-Item -ItemType Directory -Name pages

Verificació:

BASH 
ls src/app/    # Linux/macOS
dir src\app    # Windows

Resultat: Veieu les 4 carpetes noves: components/, services/, models/, pages/.

5.7. Pas 5: Crear fitxers .gitkeep

Per mantenir les carpetes buides a Git:

BASH 
# Linux/macOS/Git Bash
touch src/app/components/.gitkeep
touch src/app/services/.gitkeep
touch src/app/models/.gitkeep
touch src/app/pages/.gitkeep

# Windows PowerShell
New-Item src\app\components\.gitkeep
New-Item src\app\services\.gitkeep
New-Item src\app\models\.gitkeep
New-Item src\app\pages\.gitkeep

Verificació: Cada carpeta preparatòria conté un fitxer .gitkeep.

5.8. Troubleshooting

Problema Causa Solució
ng new falla amb error de permisos Permisos insuficients Linux/macOS: NO useu sudo. Canvieu ownership: sudo chown -R $USER:$USER ~/workspace. Windows: Executeu PowerShell com a administrador.
npm ERR! network durant instal·lació Connexió internet Verifiqueu connexió. Proveu npm config set registry https://registry.npmjs.org/.
Instalació triga més de 10 minuts npm lent o cache corrupte Cancel·leu (Ctrl+C) i proveu npm cache clean --force abans de tornar a executar ng new.
Carpetes preparatòries no es veuen a Git .gitignore massa restrictiu Assegureu-vos que .gitkeep està dins de cada carpeta buida.

5.9. Checkpoint

Projecte cataleg-elements generat amb ng new
Estructura amb src/app/, angular.json, package.json visible
node_modules/ instal·lat (200-400 MB)
Carpetes preparatòries creades: components/, services/, models/, pages/
Fitxers .gitkeep dins de cada carpeta preparatòria
Projecte obert a VS Code sense errors

6.9. Pas 7: Provar hot reload amb canvis ràpids

Feu 3 canvis consecutius a app.component.html (un cada vegada, desant després de cada):

  1. Afegiu <h2>Subtítol de prova</h2> després del paràgraf
  2. Canvieu el text del paràgraf a "El hot reload funciona perfectament!"
  3. Afegiu <p>Canvi número 3</p>

Verificació: Després de cada canvi, el navegador actualitza automàticament en 1-2 segons sense necessitat de prémer F5.

6.10. Troubleshooting

Problema Causa Solució
Port 4200 is already in use Servidor previ executant-se Trobeu el procés: lsof -i :4200 (Linux/macOS) o netstat -ano | findstr :4200 (Windows) i atureu-lo, o useu: ng serve --port 4201.
Hot reload no funciona Watch mode no actiu Atureu servidor (Ctrl+C) i proveu: ng serve --poll=2000.
Canvis HTML no es reflecteixen Cache navegador Premeu Ctrl+Shift+R (hard refresh) al navegador.
console.log no apareix Consola filtrada A DevTools, assegureu-vos que el filtre és "All levels" (no només Errors).

6.11. Checkpoint

ng serve arrenca sense errors
Aplicació accessible a http://localhost:4200
Canvis a HTML es reflecteixen automàticament (hot reload)
Canvis a TypeScript compilen i executen correctament
console.log apareix a la consola del navegador
Estils SCSS s'apliquen correctament

Abans de continuar: Deixeu el servidor executant-se.

7.12. Troubleshooting

Problema Causa Solució
git push falla amb error 403 Credencials incorrectes Creeu token PAT a GitHub (Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token). Àmbit: repo. Copieu el token i useu-lo com a contrasenya.
git remote add diu "remote origin already exists" Remote ja configurat Elimineu amb git remote remove origin i torneu a afegir-lo.
Fitxers molt grans node_modules/ sense ignorar Verifiqueu que .gitignore conté /node_modules/. Si va pujar, elimineu-lo del repositori: git rm -r --cached node_modules/ i torneu a fer commit.
Error "updates were rejected" Branch local i remot divergeixen Primera vegada: git push -u origin main --force (NOMÉS si el repositori remot està buit).

Problemes transversals

Símptoma On passa Causa Solució
Comandes npm molt lentes Terminal, npm install Cache npm corrupte npm cache clean --force i torneu a instal·lar
VS Code no detecta extensions Angular Editor, fitxers .html Extension no activa Reload window: Ctrl+Shift+P → "Developer: Reload Window"
Git demana credencials contínuament Terminal, git push Sense cache credencials git config --global credential.helper cache (Linux/macOS) o wincred (Windows)
Hot reload intermitent Navegador Sistema fitxers lent ng serve --poll=2000 per forçar polling cada 2s

Error Causa Solució Windows Solució Linux/macOS
PATH no actualitzat Instal·lació recent Reinicieu PowerShell Feu source ~/.bashrc o source ~/.zshrc
Permisos insuficients npm Carpeta global protegida Executeu PowerShell com admin Canvieu ownership: sudo chown -R $USER /usr/local/lib/node_modules
Port 4200 ocupat Altra app executant-se netstat -ano | findstr :4200, després aturar procés lsof -i :4200, després kill -9 PID
