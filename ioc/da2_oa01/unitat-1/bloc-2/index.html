1.2.1. Què és un IDE

Un IDE (Integrated Development Environment) és una aplicació que integra múltiples eines en un sol lloc: editor amb ressaltat de sintaxi, depurador, terminal, control de versions i extensions.

1.2.2. Per què necessitem un IDE

Problema: Escriure codi amb editor de text pla (Notepad, TextEdit) no detecta errors, no suggereix completacions, no integra Git, no executa comandes.

Solució: Un IDE modern com VS Code ofereix:

  • Autocompletat intel·ligent
  • Detecció d'errors en temps real
  • Terminal integrada
  • Control de versions (Git)
  • Extensions per frameworks específics

1.2.4. Quan usar un IDE complet vs editor lleuger

IDE complet (WebStorm, IntelliJ):

  • Projectes grans empresarials
  • Equips amb pressupost
  • Refactorització complexa
  • Depuració avançada

Editor extensible (VS Code):

  • Projectes de tot tipus (petits a grans)
  • Gratuït i de codi obert
  • Lleuger i ràpid
  • Extensible amb plugins
  • Recomanat per aquest curs

1.3.2. Conceptes clau

  • Editor com a nucli: VS Code és el punt central on escrivim codi, obrim fitxers i gestionem el projecte
  • Extensions com a capes: Afegeixen funcionalitats específiques sense sobrecarregar l'aplicació base
  • Terminal integrada com a pont: Permet executar npm, ng, git sense sortir de l'editor
  • Configuració multiplataforma: Les mateixes extensions funcionen a Windows, macOS i Linux
  • PATH global: Comanda code accessible des de qualsevol terminal per obrir fitxers/carpetes
  • IntelliSense: Sistema d'autocompletat que entén el context del codi TypeScript/JavaScript
  • Format On Save: Prettier s'executa automàticament en desar (Cmd/Ctrl+S)

  • Recàrrega automàtica en desar

2.2.4. Què és Git

  • Distribuït: Cada ordinador té l'historial complet
  • Branques lleugeres: Crear branca = instant, sense espai extra
  • Integritat: Hash SHA-1 garanteix no corrupció
  • Estàndard: Linux, VS Code, Angular, React, milions de projectes

2.3.3. Conceptes clau

  • Directori de treball: Carpeta visible amb fitxers editables. Canvis aquí no estan guardats a Git.
  • Stage (Staging Area): Zona intermèdia. Fitxers aquí estan preparats per commit.
  • Repositori: Carpeta .git/ amb tot l'historial. Commits aquí són permanents.
  • Commit: Instantània immutable amb autor, data, missatge i hash únic (SHA-1).
  • Branch: Línia de desenvolupament. Defecte: main (abans master).
  • HEAD: Punter indicant commit/branca actual.
  • Distribuït: Cada ordinador té còpia completa de l'historial. No depèn de servidor central.

2.3.4. Metàfora: Git com a màquina del temps

Git és una màquina del temps per al codi:

  • Cada commit = foto del projecte en un moment
  • git log = àlbum de fotos cronològic
  • git checkout = viatjar a una foto anterior
  • Branca = línia temporal alternativa (multiversos!)

  • Repositori local: Còpia del projecte amb historial al nostre ordinador (carpeta amb .git).
  • Repositori remot: Còpia del projecte al servidor accessible per Internet (GitHub, GitLab, Bitbucket).
  • GitHub: Plataforma d'allotjament de repositoris Git més popular, propietat de Microsoft.
  • Origin: Nom per defecte del repositori remot principal.
  • Comandes remotes:
    • git remote add origin <URL>: Connecta repositori local amb remot.
    • git remote -v: Llista remots configurats.
    • git push -u origin main: Publica commits locals al remot primera vegada (configura tracking).
    • git push: Publica commits locals al remot (després de tracking).
    • git pull origin main: Baixa commits del remot i fusiona amb local.
    • git clone <URL>: Crea còpia local completa d'un repositori remot.
  • Personal Access Token (PAT): Token d'autenticació per accedir GitHub des de terminal (substitueix contrasenya).
  • Repositori públic: Visible per tothom, ideal per open source i portfolis.
  • Repositori privat: Només visible per vós i convidats, ideal per projectes confidencials.
  • Tracking: Relació entre branca local i remota que permet git push sense especificar remot.
  • Upstream: Branca remota de la qual fa seguiment una branca local.
  • Fork: Còpia d'un repositori remot al vostre compte GitHub (per contribuir a projectes d'altres).
  • Clonació: Crear còpia local completa d'un repositori remot, incloent tot l'historial.
